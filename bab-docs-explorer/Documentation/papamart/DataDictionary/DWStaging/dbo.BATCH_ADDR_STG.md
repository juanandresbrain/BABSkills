# dbo.BATCH_ADDR_STG

**Database:** DWStaging  
**Server:** papamart  

## Columns

| Column | Type | Max Length | Nullable | PK | FK | Description |
|---|---|---|---|---|---|---|
| STG_ID | int | 4 | 0 |  |  | The identifier of the record in the CRM_STG or KSK_REGIS_STG table.  The STG_DTA_SET_CD field should be used in order to determine whether this represents a record from the CRM_STG table or the KSK_REGIS_STG table.  If this is from the CRM_STG table, the STG_DTA_SET_CD will be set to CRM.  If this is from the KSK_REGIS_STG table, the STG_DTA_SET_CD will be set to Kiosk.  This STG_ID is used to tie together all of the identifiers associated with a particular staging record so that it is easier to process records during the loads into the Data Warehouse. |
| STG_DTA_SET_CD | varchar | 20 | 0 |  |  | This field identifies from which staging table this record is sourced.  If this record comes from the CRM_STG table, this will be set to CRM.  If this record comes from the KSK_REGSI_STG Table, this will be set to Kiosk.  This is needed in order to help uniquely identify records in the table, since it is possible that each staging table may contain the same unique identifier (CRM_STG_ID and KSK_REGIS_STG_ID). |
| ADDR_LN_1_TXT | varchar | 60 | 1 |  |  | The first line of the address for the customer, returned from the Address Cleansing process, based on data entered into the kiosk at the time of animal registration.  This typically contains at least the street number and street for the address. |
| ADDR_LN_2_TXT | varchar | 60 | 1 |  |  | The second line of the address for the customer, returned from the Address Cleansing process, based on data entered into the kiosk at the time of animal registration.  This may contain Apartment, Suite or other more detailed information necessary to locate the address. |

| ADDR_LN_5_TXT | varchar | 60 | 1 |  |  | The fifth line of the address for the customer, as returned from the Address Cleansing process, based on data entered into the kiosk at the time of animal registration.  This is not populated very often. |

For Canadian addresses, this will be an abbreviation for one of the Canadian provinces.  For some international countries, this may be blank.    Cities, Counties and Postal Codes belong to State/Provinces.  State/Provinces belong to Countries. |
| ST_PRVNC_NM | varchar | 60 | 1 |  |  | The full name for the State or Province in which the address resides, returned by the address cleansing process, based on the data entered into the kiosk at the time of animal registration or into the CRM system.  For US addresses, this will be the full name of one of the 50 states.  For Canadian addresses, this will be the full name of one of the Canadian provinces.  For some international addresses, this will be blank.    Cities, Counties and Postal Codes belong to State/Provinces.  State/Provinces belong to Countries. |
| PSTL_CD | varchar | 10 | 1 |  |  | The postal code in which this address resides, as returned from the Address Cleansing process, based on the postal code entered into the kiosk at the time of animal registration or into the CRM system.  Postal Codes are used by the postal service to group address together for mail routing and delivery.  In the United States, there is usually a relationship between a postal code and a post office.  Cities and Counties are grouped into Postal Codes, which are grouped into States/Provinces. |
| PSTL_PLS_4_CD | varchar | 5 | 1 |  |  | The additional 4 digits that are added to US postal codes to classify address at a lower level than the Postal Code.  These four digits break each postal code down into smaller sections, more accurately identifying the location of a particular address within the postal code. |
| CNTRY_ABBRV | varchar | 5 | 1 |  |  | The abbreviation for the country in which this address resides.  Most of these values are two character country codes, however, there are some that are three characters or more.  States/Provinces belong to countries, which are the highest part of the geographical hierarchy for addresses. |
| CNTRY_NM | varchar | 60 | 1 |  |  | The full name of the country in which this address resides.  States/Provinces belong to countries, which are the highest part of the geographical hierarchy for addresses. |
| CARI_RTE_CD | varchar | 10 | 1 |  |  | A code representing the carrier route used by the postal service when delivering mail. |
| LAT_NBR | varchar | 20 | 1 |  |  | The number that represent the latitude at which this address resides.  This is defined by the address cleansing process and is used, along with longitude to calculate the distance to nearest store for the address.   Latitude represents a location on the globe, in relation to the equator.  Latitudes are given in northern latitude (north of the equator) and southern latitude (south of the equator).  Latitudes are represented in terms of degrees, minutes and seconds. |
| LONG_NBR | varchar | 20 | 1 |  |  | The number that represent the longitude at which this address resides.  This is defined by the address cleansing process and is used, along with latitude to calculate the distance to nearest store for the address.   Longitude represents a location on the globe, in relation to the prime meridian located near the old Royal Astronomical Obseravtory in Greenwich, England.  A meridian is one of 360 imaginary lines that stretch from the North Pole to the South Pole.  Longitudes are represented in terms of degrees, minutes and seconds. |
